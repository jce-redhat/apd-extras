# Project Development Workflow

Review the jobs.md file for any jobs that are not marked as "Complete" or "Needs Info".  For each of these:

1. Review the job specification file for requirements that are unclear or incomplete.  Analyze the requirements and determine if there are missing details that may be needed before the spec can be implemented.
2. If the specification is unclear, update the job with a succinct list of items that require feedback, then mark the job as "Needs Info" and move to the next job.
3. Create a new feature branch and implement the job.  No human input is required to begin implementing jobs with clear specifications.
4. Update the job specification file and mark the job as complete.
